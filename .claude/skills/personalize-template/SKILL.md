# Personalize Template Skill

This skill takes a user-supplied email template with `[Placeholder]` tags,
personalizes it for one or more companies using `template_personalizer.py`,
and returns the completed emails ready to review or send.

## Working directory and Python path
- **Working directory:** `C:\Users\admin\linkedin-scraper`
- **Python:** `.\venv\Scripts\python.exe`

## What you need from the user
Gather before proceeding:
1. **The template** — the raw email body with `[Placeholder]` tags. If the
   user pastes it inline, use that. If they reference a file, read it.
2. **Company + contacts** — for each company: name, domain, industry, and a
   list of contacts (first_name, last_name, title). If not provided, check
   the database for existing contacts.
3. **LLM preference** — default to Anthropic with Perplexity fallback
   (already handled automatically by `ClaudeClient`).

## Placeholder conventions
Recognised static tags (filled from data, no LLM needed):
- `[First Name]` → contact's first name
- `[Last Name]` → contact's last name
- `[Company Name]` → company name
- `[Title]` → contact's title

Sender placeholder (left untouched — filled by the human before sending):
- `[Your Name]`

Dynamic tags (LLM generates 1–2 sentences for each):
- `[Personalization for company]`, `[Custom hook]`, `[Relevant insight]`,
  or any other descriptive phrase inside brackets

## Running the personalizer

Write and run a short inline script. Example pattern:

```python
import sys, os
sys.stdout.reconfigure(encoding='utf-8', errors='replace')
os.chdir(r'C:\Users\admin\linkedin-scraper')

from template_personalizer import personalize_for_company, analyze_placeholders
from claude_client import ClaudeClient

template = """<PASTE TEMPLATE HERE>"""

company = {"name": "Stripe", "domain": "stripe.com", "industry": "Fintech"}
contacts = [
    {"first_name": "Patrick", "last_name": "Collison", "title": "CEO"},
    {"first_name": "John",    "last_name": "Collison", "title": "President"},
]

# Show what placeholders were detected
info = analyze_placeholders(template)
print(f"Static: {info['static']}")
print(f"Dynamic (LLM): {info['dynamic']}")
print()

claude = ClaudeClient()
results = personalize_for_company(template, contacts, company, claude)

for r in results:
    c = r['contact']
    print(f"=== {c['first_name']} {c['last_name']} ({c['title']}) ===")
    print(f"Subject: {r['subject']}")
    print(r['body'])
    print()
```

Run with:
```bash
.\venv\Scripts\python.exe -u -c "<script above>" 2>&1
```

Or save as a temp file and run it if the template is long.

## Loading contacts from the database
If the user says "use the contacts already in the DB for company X":

```python
import sqlite3
conn = sqlite3.connect('outreach.db'); conn.row_factory = __import__('sqlite3').Row
contacts_raw = conn.execute("""
    SELECT ct.first_name, ct.last_name, ct.title
    FROM contacts ct JOIN companies co ON ct.company_id=co.id
    WHERE co.name=? AND ct.primary_email IS NOT NULL
    ORDER BY ct.created_at LIMIT 10
""", ('CompanyName',)).fetchall()
contacts = [dict(r) for r in contacts_raw]
conn.close()
```

## What `personalize_for_company` does
1. **`analyze_placeholders`** — classifies every `[Tag]` as static or dynamic
2. **`fill_static_placeholders`** — fills data-driven tags with no LLM
3. **LLM call (once per company)** — `ClaudeClient.personalize_from_template()`
   fills dynamic tags for the first contact
4. **`clone_for_contact`** — word-boundary-safe name/title swap for all other
   contacts at the same company (no extra LLM calls)

Returns: `[{"contact": {...}, "subject": "...", "body": "..."}]`

## Reporting to the user
Show each personalized email inline, clearly separated by contact name. Then:
- Highlight which dynamic placeholders were generated by the LLM
- Note the subject line generated for each (or confirm `[Your Name]` is still
  present for the sender to fill)
- If the user wants to send these emails, tell them to run `outreach-campaign`
  or add the results to `send_records` manually

## Failure handling
- **`[Personalization for company]` not replaced** → check the template uses
  exact bracket syntax `[...]`; run `analyze_placeholders(template)` to confirm
  it shows up in `dynamic`
- **LLM returns plain text instead of JSON** → `personalize_from_template`
  handles this gracefully — it returns the raw text as the body
- **Contact dict missing `first_name`** → use `firstName` as fallback key;
  both formats are supported
- **`ClaudeClient` credits exhausted** → Perplexity fallback is automatic;
  watch for `[LLM fallback] used perplexity` in output
